<span data-ttu-id="68445-121">Antes de usar uma propriedade de FCI do Windows Server ou outra propriedade em uma política de DLP, você precisa criar uma propriedade gerenciada no centro de administração do SharePoint.</span><span class="sxs-lookup"><span data-stu-id="68445-121">Before you can use a Windows Server FCI property or other property in a DLP policy, you need to create a managed property in the SharePoint admin center.</span></span> <span data-ttu-id="68445-122">Veja por quê.</span><span class="sxs-lookup"><span data-stu-id="68445-122">Here's why.</span></span>
  
<span data-ttu-id="68445-p107">Exemplos</span><span class="sxs-lookup"><span data-stu-id="68445-p107">In SharePoint Online and OneDrive for Business, the search index is built up by crawling the content on your sites. <span data-ttu-id="68445-p108">Isso é importante porque a DLP no Office 365 usa o rastreador de pesquisa para identificar e classificar informações confidenciais em seus sites e, em seguida, armazenar informações confidenciais em uma parte segura do índice de pesquisa. Mas, para usar uma FCI ou outra propriedade em uma política de DLP, essa propriedade rastreada precisa ser mapeada para uma propriedade gerenciada para que o conteúdo com essa propriedade seja mantido no índice.</span><span class="sxs-lookup"><span data-stu-id="68445-p108">This is important because DLP in Office 365 uses the search crawler to identify and classify sensitive information on your sites, and then store that sensitive information in a secure portion of the search index. But to use an FCI or other property in a DLP policy, that crawled property needs to be mapped to a managed property so that content with that property is kept in the index.</span></span>
  
<span data-ttu-id="68445-133">Para obter mais informações sobre propriedades gerenciadas e pesquisa, consulte [gerenciar o esquema de pesquisa no SharePoint Online](http://go.microsoft.com/fwlink/p/?LinkID=627454).</span><span class="sxs-lookup"><span data-stu-id="68445-133">For more information on search and managed properties, see [Manage the search schema in SharePoint Online](http://go.microsoft.com/fwlink/p/?LinkID=627454).</span></span>
