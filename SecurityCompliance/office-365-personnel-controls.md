# <a name="office-365-personnel-controls"></a>Controles de pessoal do Office 365 

Pessoal de triagem, que é o processo de revisão e validando últimos comportamento e o status de uma pessoa, é um controle de redução de risco importantes para impedir o comprometimento de serviço do Office 365. Comportamento últimos não está perfeito previsão da comportamento futuras de uma pessoa, ele ajudar a identificar possíveis atores de bad. Padrão de triagem do pessoal da Microsoft se aplica a todos os funcionários, estagiários e equipe contingente envolvida no desenvolvimento, a operação ou o fornecimento de serviços online para o governo ou clientes de nuvem comercial Microsoft. Ambientes que não são operados pela Microsoft triagem padrões para nuvem nacional são definidos pelo pessoal de parceiro operacionais para cada ambiente específico.

## <a name="microsofts-personnel-screening-standard"></a>Equipe da Microsoft triagem Standard

A equipe da Microsoft triagem práticas recomendadas para o Office 365 estejam alinhada às controles Instituto Nacional de padrões e tecnologia (NIST) e padrões corporativos da Microsoft para a equipe de triagem. Equipe da Microsoft que exige acesso ao seguinte estará sujeito pessoal triagem padrão da Microsoft:
- Acesso físico à data centers, colegas locais, salas protegidas, compartimentos, gabinetes de servidor ou sites de borda que fornecem a infra-estrutura de suporte a serviços online para clientes de nuvem comercial ou governamentais.
- Acesso lógico para o governo ou nuvem comercial fornecidas em ambientes gerenciados específicos de dados do cliente.
    - Acesso de gerenciamento aos serviços de transporte ou armazenam governo ou nuvem comercial dados do cliente e dispositivos de rede.

Os eventos específicos relacionados a equipe que acionam a triagem requisitos incluem:
- Nova equipe Microsoft colocados em funções onde a triagem é um requisito definido.
- Equipe interna da Microsoft transferindo ou mover para uma função existente que atualmente inclui triagem como um requisito definido.
- Funções existentes que alteram o escopo para incluir a triagem como um requisito definido.

## <a name="screening-enforcement-criteria"></a>Imposição de critérios de classificação

Para garantir que somente o pessoal aprovado tenha acesso aos dados do cliente ou em ambientes que requerem a triagem, os seguintes critérios de imposição aplicável:

**Estados Unidos ambientes somente em nuvem**:
- Acesso a dados do cliente ou em ambientes que requerem a triagem deve ser permitida somente depois adjudicação for concluída e requisitos de triagem são passados.
- Equipe da Microsoft que não mais exige acesso a dados do cliente ou ambientes relacionados deve ter acesso removido após deixando Microsoft ou mover para uma nova função.
- Equipe da Microsoft deve deixar cartões inteligentes de ambiente filtrada com gerenciamento antes de sair dos Estados Unidos.

**Ambientes de nuvem nacional**:
- Pessoal de objeto de confiança de operador ou dados de terceiros é responsável pelo gerenciamento e aplicação de acesso para ambientes de nuvem nacional.

Em ambientes de serviços de nuvem da Microsoft, acesso é restrito com base na função de uma pessoa e o tipo de dados envolvidos, conforme detalhado na tabela a seguir. Pessoal qualificado ou não qualificado fisicamente localizado fora dos Estados Unidos não têm permissão para ter acesso aos dados de cliente em uma nuvem Estados Unidos. Acesso para ambientes de nuvem nacional é restrito para que a equipe da Microsoft não têm técnico acesso a dados do cliente ou sistemas que contêm dados do cliente, sem a aprovação pelo objeto de confiança do terceiro operador ou dados.

| Função | Acesso aos dados do cliente | Acesso aos dados do sistema |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------|---------------------------------|
| Pessoal qualificado fisicamente localizado nos Estados Unidos | Permitido | Permitido |
| Pessoal qualificado fisicamente localizado fora dos Estados Unidos | Não permitido | Permitido |
| Acesso de exceção internacional para a Microsoft Staff – permite acesso lógico para a equipe da Microsoft que não residem no país em que o governo ou dados comerciais do cliente está em repouso | Não permitido | Permitido |
| Equipe não qualificada (equipe não projetadas e que exigem um escort pela equipe qualificado) | Permitida com autorização | Permitida com supervisão escort |


## <a name="microsoft-pre-employment-screening"></a>Triagem de emprego de pré-lançamento do Microsoft

Onde leis locais permite a ele, departamento de segurança da Microsoft Global conduz triagem de emprego prévia. Esta é uma investigação de plano de fundo formal que inclui os seguintes critérios:
- Uma verificação (por exemplo, para integridade e a precisão) de currículo do candidato
- Confirmação de qualificações acadêmicas e profissionais
- Investigação de qualquer perda de credenciais profissionais
- Verificação de três empregadores últimos
- Uma verificação de registros de polícia de condenações felony
- Confirmação da identidade de uma identificação emitido pelo governo
- Seleção de crédito onde apropriado

Rescreening periódica e/ou plano de fundo adicional verificações podem ser necessária para determinadas gerenciamento, segurança ou outras funções, incluindo, mas não limitado aos funcionários baseados nos Estados Unidos nas funções que exigem acesso aos dados do cliente. Para a equipe contingente, o contrato com o terceiro especifica requisitos da Microsoft para triagem que deve ser realizado por terceiros. Para verificações de plano de fundo, a empresa de terceiros é responsável por fornecer a verificação da Microsoft que tenha sido executada a uma verificação de plano de fundo. Os resultados da verificação de plano de fundo geralmente são recebidos via email do departamento de recursos humanos de terceiros. Funcionários internacionais da equipe de contrato podem ser isentos do plano de fundo triagem processo devido às leis em países que proíbem verificações em segundo plano.

## <a name="microsoft-employment-screening"></a>Triagem de emprego da Microsoft
Desde 2004, a Microsoft possui indivíduos necessários para passar a uma tela de registro de sete anos criminais para felonies e misdemeanors e verificar suas educação e histórico de emprego, como parte de emprego de pré-lançamento dos funcionários e estagiários de triagem nos Estados Unidos.

Nos Estados Unidos, antes de atribuir qualquer funcionário da Microsoft ou qualquer subcontratado atribuído pelo Microsoft para fornecer serviços relacionados ao Office 365, Microsoft realizará e fazer com que o seu subcontratado para realizar uma verificação de plano de fundo, consistindo em um Seguridade Social número de rastreamento e criminoso registram a seleção. Os dados dessa verificação de plano de fundo são usados como um fator para a decisão de contratação. A verificação de registro criminais inclui uma sete anos felony misdemeanor registros criminais verificação e dos registros federais, estaduais e county (conforme aplicável).

Como uma condição de emprego, no momento da contratação, todos os funcionários da Microsoft são necessários para assinar confidencialidade e não divulgação e verificar que eles revisou o manual de funcionários da Microsoft.

## <a name="microsoft-cloud-background-check"></a>Seleção de plano de fundo de nuvem da Microsoft
Um plano de fundo de nuvem da Microsoft seleção é necessária para candidatos ser contratado como funcionários de fornecimento de serviços relacionados ao Office 365 nos Estados Unidos. Os funcionários da Microsoft nos Estados Unidos com acesso aos dados do cliente devem seguir o processo de seleção de plano de fundo do Microsoft Cloud, que é exigido por todos os serviços do Office 365. Fora dos Estados Unidos, o processo varia. Por exemplo, o Microsoft Cloud para a Alemanha usa um modelo de aprovação de objeto de confiança de dados, que foi projetado para garantir que o objeto de confiança de dados (uma empresa alemão) e não com a Microsoft, seja no controle de acesso a dados do cliente. O Microsoft Cloud na Alemanha é entregue de data centers na Alemanha, e os centros de operações (OC) contendo a equipe técnica do objeto de confiança dados também estão na Alemanha. Tanto o datacenter como as instalações do OC são operadas, mantidas e controladas pelo objeto de confiança de dados.

A tabela a seguir lista as verificações executadas como parte da verificação de plano de fundo do Microsoft Cloud.

| Triagem | Descrição |
|--------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| Pesquisa de número de seguridade social | Verifica se o número de Seguridade Social fornecido é válido. |
| Verificação de antecedentes criminais | Registros de sete anos criminais verificar offenses felony e misdemeanor no locais, state e county níveis e conforme apropriado, no nível federal. |
| Office da lista de controle de ativos estrangeira | Lista de departamento do Tesouro de pessoas e organizações com as quais cidadãos dos Estados Unidos e residentes permanentes não são permitidos para fazer negócios. |
| Bureau do setor e lista de segurança | Lista de departamento de comércio de indivíduos e entidades bloqueadas de participar de atividades de exportação. |
| Office de defesa comerciais controles jurídicas pessoas de lista (*adicionado em 1 de julho de 2010*) | Lista de departamento de estado dos indivíduos e entidades bloqueadas de participando em Exportar atividades relacionadas no setor de defesa. |


Os resultados da verificação de plano de fundo do Microsoft Cloud são armazenados em nosso banco de dados do funcionário, que é conectado ao nossos sistemas de controle de acesso do datacenter. Se a verificação de plano de fundo do Microsoft Cloud expirar e o funcionário não renovar, em seguida, acesso aos serviços do Office 365 é revogadas e não estão mais disponíveis até que a verificação de plano de fundo do Microsoft Cloud seja concluída novamente. Quando o relacionamento de emprego com a Microsoft for encerrada, nenhum acesso existente do datacenter imediatamente será revogado.

Ações de sociais Estados Unidos é verificada para todos os funcionários com acesso físico ou lógico aos serviços do governo do Office 365 dos Estados Unidos. Para verificar as ações sociais, funcionários e/ou novo candidatos de contratação se encontrar com um representante de ações sociais americano treinados para examinar a documentação verificando ações sociais de US. Funcionários ou novo candidatos de contratação devem trazer a documentação necessária e assinar um formulário de certificação em uma reunião com o delegado de ações sociais para sua região. A reunião deve ser realizada em pessoa. Depois que o indivíduo tem atendidos com o delegado de ações sociais e fornecidos a documentação necessária e assinaturas, o representante de ações sociais encaminha uma cópia dos documentos para o Microsoft Operations pessoal que enviam a cópia de manutenção de registros.

Pessoal com acesso lógico para a nuvem de comunidade governamental dos EUA do Office 365 ou acesso físico ou lógico para ofertas governamentais do Azure EUA, é necessários para cumprir os requisitos do governo federal do FBI [justiça Criminal informações Serviços](https://www.fbi.gov/services/cjis) (CJIS), incluindo a triagem de pessoal. A triagem CJIS para oferecer suporte o serviço Office 365 US governamentais inclui uma verificação de impressão digital com base no plano de fundo criminais que é aquelas subordinadas pela agência de sistema dos CJIS designada adjudicator nos [estados que tenham sido registrados](https://blogs.office.com/2013/10/23/california-and-microsoft-sign-cjis-security-policy-agreement/) em Microsoft Online Serviços CJIS programa de suporte.
